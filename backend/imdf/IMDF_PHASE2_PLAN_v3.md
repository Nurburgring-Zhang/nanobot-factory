# IMDF 第二阶段执行计划 v3.0 — 三Agent互审产出

## 三Agent分析汇总

### Agent A (架构) — 依赖与风险
依赖链: 优雅关闭(无依赖) → 数据库迁移(无依赖) → 数据导入(迁移后) → 增量交付(导入后)
       → AI预标注(无依赖) → 缩略图预览(无依赖) → 管道监控(队列就绪)
       → 数据浏览器(后端API就绪后) → 运营看板(统计API就绪后)
       → 节点化工作流引擎(所有就绪后)

风险评级:
- 高: 节点化工作流引擎(DAG正确性+并发控制)
- 中: AI预标注(API依赖), 缩略图(格式兼容), 数据导入(Excel兼容)
- 低: 其余6项

### Agent B (产品) — 价值/难度矩阵
高价值低难度(立即做): 优雅关闭, 缩略图预览
高价值中难度: AI预标注, 数据浏览器, 运营看板
中价值低难度: 数据库迁移, 增量交付, 管道监控
中价值中难度: 数据导入
高价值高难度: 节点化工作流引擎(最后做)

### Agent C (工程) — 代码量估算
大(750-1250行): 数据导入, 节点化工作流引擎
中(400-700行): AI预标注, 数据浏览器, 运营看板, 缩略图, 管道监控
小(100-350行): 增量交付, 优雅关闭, 数据库迁移
总计: ~5,400行新增代码
