## 5. 关键设计决策

1. **三层兜底链** (data-collection.js): API 真值 → 客户端已加载数据求和 → `—` 显示。永不 Math.random 兜底, 符合"不能前端瞎造"原则。

2. **后端端点最小化** (operators/stats): 复用现有 `engines/operators_lib.get_registry()`, 不重新维护一份算子列表。单点真值 (single source of truth)。

3. **aesthetic_health 扩展而非新端点** (max_srcc): 复用现有 health 检查, 减少 router 数量。前端 0 行额外网络请求。

4. **data-viewer.js 真实缩略图 fallback**: API 返回 `preview_url/thumbnail_url` 时用 `<img>` (真实数据), 否则用 emoji 占位。**不用 picsum.photos** (符合 R4 final gate 要求)。

5. **picture-book.js 复用 bk-status DOM**: 原本 L31 已有空 div, AI 失败时直接写 textContent, 不新增 DOM, 不破坏既有布局。

6. **CSS style 持续 24px emoji grid thumb**: gridItem 缩略图统一 120px 高度, `<img>` object-fit:cover, emoji flex 居中 32px, 视觉一致。

7. **sanitizeHTML 全覆盖**: zhiying.js (需求标题/状态)、data-viewer.js (名称/标签/类型) 都加 `sanitizeHTML`, 防止 XSS 注入。
