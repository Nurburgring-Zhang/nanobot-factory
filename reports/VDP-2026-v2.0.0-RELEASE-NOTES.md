## 10. Migration Notes (v1.5.0 → v2.0.0)

### Breaking changes
- **Alembic chain**: legacy chain marked DEPRECATED but kept (test DBs depend on it). New chain starts at 0001-0007. New migrations should go in `backend/imdf/alembic/versions/`.
- **Skill Pydantic models**: 16 files have `from __future__ import annotations` REMOVED. If you re-add this import, models will fail at instantiation.
- **Frontend locale loading**: `LOCALE_META`, `RTL_LOCALES`, `isRTL`, `applyDocumentDirection` exports added to `frontend-v2/src/locales/index.ts`. Update imports if you have direct locale file access.

### Config changes
- New env var: `CORS_ALLOW_ORIGINS` (comma-separated origins, default `http://localhost:5173,http://localhost:8765`)
- `.env.production` no longer tracked in git (was `R` per git status)
- MFA is now enforced on login (was previously bypassed)

### No required user actions
- v2.0.0 is backward-compatible with v1.5.0 data and configs
- Existing projects will work without migration
