## 3. 14 FAIL (实际 17, 修后 0) 根因分类

### 3.1 根因 A: 缺 `argon2-cffi` + passlib 1.7/bcrypt 5.x 兼容 (影响 14 用例)

**现象**: 所有 login / refresh / GDPR 测试返回 401 "Invalid username or password", 警告日志:
```
WARNING  api.auth_routes:auth_routes.py:81 bcrypt 72-byte truncation failed, falling back to SHA-256: password cannot be longer than 72 bytes
WARNING  api.auth_routes:auth_routes.py:105 passlib verify failed: hash could not be identified
WARNING  api.auth_routes:auth_routes.py:99 argon2 verify failed: <empty>
```

**根因链**:
1. `argon2-cffi` 未装 → `_password_backend = "unknown"` 路径走不通
2. 降级到 passlib (`_password_backend = "passlib_bcrypt"`)
3. `bcrypt 5.0.0` 移除了 `__about__` 属性 → passlib 1.7.4 启动时 `bcrypt.__about__.__version__` AttributeError
4. bcrypt hash 写入失败 → fallback 到 SHA-256 (`sha256$salt$hash` 格式)
5. 后续 `_verify_password` 用 passlib CryptContext 验证 SHA-256 哈希 → "hash could not be identified" → False → login 401

**修复**: 装 `argon2-cffi 25.1.0`, auth_routes.py L44-48 优先级最高, 自动接管 hash + verify。

### 3.2 根因 B: `reset_security_state_for_tests()` 不清 `users_db` (影响 2 用例)

**现象**: `test_revoked_cache_persisted_to_db` 和 `test_register_then_login_full_flow` 等独立 register 类测试报 400 "Username already exists"。

**根因**: fixture `reset_security_state_for_tests()` 旧实现只清 `_revoked_cache` 和重置 RateLimiter, **不**清 `users_db` 内存字典; 但 DB 中已有上次跑剩的 `alice_r95` 记录 (用旧 SHA-256 hash 写入的)。register 检测 `username in users_db` (内存) → 之前测试残留导致 400。

**修复**:
```python
def reset_security_state_for_tests() -> None:
    global _revoked_cache, users_db
    _revoked_cache = set()
    users_db = {}  # 清空内存, register 时会重写 + INSERT OR REPLACE 进 DB
    rate_limiter.reset()
    # 不再调 _load_users() — 让测试完全从内存起步, 避免上一次 DB 残留
```

### 3.3 根因 C: `CSRFMiddleware.logger.warning()` 用 kwargs 传非标准字段 (影响 1 用例)

**现象**: `test_csrf_missing_token_returns_403` 抛 `TypeError: Logger._log() got an unexpected keyword argument 'method'`。

**根因**: Python 标准 `logging` 库的 `Logger._log(level, msg, args, exc_info=None, extra=None, ...)` 只接受 `exc_info` / `extra` / `stack_info` / `stacklevel` 这几个 kwargs; 但代码用了 `logger.warning("csrf_origin_blocked", method=method, path=path, origin=origin)` 把 `method` / `path` / `origin` 当 kwargs 传 — 标准库不接受, 抛 TypeError, 中间件崩溃。

**修复**: 改成 f-string 形式, 不依赖 kwargs:
```python
logger.warning(
    f"csrf_origin_blocked method={method} path={path} origin={origin}"
)
```

### 3.4 根因 D (轻度, 已被根因 A 修复): `Logger._log() got an unexpected keyword argument 'method'` 在 security_middleware.py L196 触发

同上 3.3, 但出现在第二个 warning 位置 (csrf_token_missing), 实际只在 `test_csrf_missing_token_returns_403` 触发 (因为 csrf_token + csrf_mismatch 测试都有 cookie/header token, 走不到 L196)。
