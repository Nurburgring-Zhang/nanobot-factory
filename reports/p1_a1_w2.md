## 3. 引擎能力

### 3.1 文本水印 — `add_text_watermark(input, output, text, position, opacity, font_size)`

- **实现**: ffmpeg `-vf drawtext` (libfreetype 字体自动检测 — Windows `arial.ttf` /
  Linux DejaVu / macOS Helvetica)
- **位置参数**: topleft / topright / bottomleft / bottomright / center
- **不透明度**: 0.0-1.0 (映射到 drawtext `fontcolor@alpha`)
- **可选文字框**: `box=1` + `boxcolor='black@0.4'` 提升可读性
- **字符转义**: 单引号 / 冒号 / 反斜杠 / 百分号自动转义

### 3.2 图片水印 — `add_image_watermark(input, output, logo, position, opacity, scale)`

- **实现**: ffmpeg `filter_complex` (scale logo → colorchannelmixer → overlay)
- **位置**: 5 个锚点同文本水印
- **不透明度**: 通过 `colorchannelmixer=aa=` 控制 alpha
- **缩放**: 默认 logo 宽度 = 视频宽度 × 15%

### 3.3 不可见水印 (音频 LSB) — `add_invisible_watermark(input, output, message)`

- **实现**: 提取音频为 PCM 16-bit → 修改 sample LSB 嵌入 `IMDF` magic + 长度 + 消息 →
  FLAC 重新封装 (无损以保留 LSB)
- **验证**: `extract_audio_watermark(path) → str` 解析回原消息
- **限制**: FLAC 容器 (而非 AAC) 是 LSB 隐写的必要条件

### 3.4 验证 — `verify_watermark(video_path, watermark_id=None) → bool`

- **有 record 时**: 提取中段帧 → SHA256 与 record 中 `frame_sha256` 比对
- **无 record 时**: 仅返回 frame 是否能成功提取 (弱校验)
- **ffmpeg 不可用**: 降级为文件存在性检查 (返回 True)
- **音频水印**: 调 `extract_audio_watermark` 比对原始 message

### 3.5 错误体系

```python
WatermarkError (base)
├── WatermarkEngineUnavailable  # ffmpeg 缺失
├── WatermarkInputError         # 输入文件 / 文本为空
└── WatermarkProcessingError    # ffmpeg 调用失败 / 损坏文件
```

## 6. 关键设计决策

### 6.1 FLAC 而非 AAC 作为音频水印的容器

AAC 是有损压缩，re-encode 会破坏 LSB。改用 FLAC (`-c:a flac`) 实现无损 roundtrip，
LSB 嵌入 + 提取可以 100% 还原原始消息。

### 6.2 Frame-level hash 而非 file-level hash

最初实现用 file SHA256 做 verify，但 re-encode 后即使是同一视频也会得到不同 hash。
改为 embed 时捕获 mid-frame SHA256，verify 时同样位置提取并比对 — 对 libx264
ultrafast CRF 26 这种确定性编码可稳定匹配。

### 6.3 帧捕获的时间戳候选

`_capture_frame_hash` 默认在 `t=1.0s` 取帧，但当视频短于 1s 时会 seek 越界导致
无帧输出。改为 duration-aware: `[time_sec, dur/2, 0.5, 0.1]` 中首个合法的。

### 6.4 Pydantic 422 而非自定义 400

position 用 `pattern=^(topleft|...|center)$` 由 Pydantic 校验 — 返回 422 比自写
if/else 鲁棒。空 text / 缺失 file 仍由 watermark engine 抛 400。

## 7. 已知限制 / 后续 TODO

1. **真实 OCR 验证**: 当前 verify 仅比对 frame hash，未做 OCR 文字识别 (需 tesseract)。
   后续可挂载 pytesseract 提取文字做严格匹配。
2. **抗裁剪**: 文本水印在视频被裁剪后会丢失。后续可考虑九宫格平铺 + 随机抖动。
3. **音频隐写对抗重采样**: 当前 LSB 在重采样后会失效。生产可考虑扩频或 echo hiding。
4. **ffmpeg 不可用时** verify 退化为「文件存在」检查 — 实际意义有限，但不会误报 False。
