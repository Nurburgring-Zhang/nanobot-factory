## 3. 设计要点

### 3.1 账单系统 (billing.py)
- **UsageMeter**: 用量事件 + 周期查询 (`events_for(tenant_id, "YYYY-MM")`)
- **TieredPricing**: 3 档 (`free` $0 / `pro` $29 / `enterprise` $299)
  - 含 included 配额 + overage 阶梯 (e.g. api_calls: 每 1000 次 5 cents)
- **InvoiceEngine**: 月度发票生成, 含 subtotal / tax / total_cents
- **存储**: `JsonlUsageStore` (生产) + `InMemoryUsageStore` (测试)
- **Decimal 计算**: 整数 cents 避免浮点误差, tax 用 `Decimal` + `ROUND_HALF_UP`

### 3.2 数据导出 (data_exporter.py)
- **JSONExporter**: 标准信封 `{schema_version, exported_at, count, meta, records}`
- **CSVExporter**: UTF-8 BOM + RFC4180 + 自动转义 (逗号/引号/换行)
- **dispatcher**: `export_data(records, fmt, ...)` 顶层入口
- **normalize**: 自动处理 dataclass / pydantic / dict 嵌套

### 3.3 审计日志 (audit_log.py)
- **hash chain**: `entry_hash = sha256(prev_hash || seq || ts || actor || action || target || payload)`
- **GENESIS_HASH**: `"0" * 64`
- **verify_chain**: 返回 `(ok, first_bad_seq)`, 任意篡改 → ok=False
- **复用 R7 logging_setup**: `get_logger("audit_log")` 写结构化日志
- **存储**: `JsonlAuditStore` (append-only) + `InMemoryAuditStore`

### 3.4 多租户 (tenant.py)
- **Tenant**: id / name / tier / enabled / quotas / metadata
- **Quota**: `hard` / `soft` / `audit` 三档
- **TenantRegistry**: CRUD + 配额管理 + JSON 持久化
- **assert_tenant_isolation**: 显式隔离校验
- **check_quota**: 返回 `QuotaDecision(allowed, level, current, limit)`
  - `hard_block` → allowed=False
  - `soft_warn` → allowed=True, 警告
  - `audit_warn` → allowed=True, 审计阈值

### 3.5 Router 整合 (r10_5_business_routes.py)
- **顶层 prefix**: `/api/v1/business`
- **4 sub-router**: billing / export / audit / tenant
- **16 endpoints** 全部挂载成功
- **共享 backend**: 模块级 `_STATE` dict (usage_store / audit_store / registry / pricing)
- **环境变量切换**: `IMDF_BUSINESS_USAGE_PATH` / `IMDF_BUSINESS_AUDIT_PATH` / `IMDF_BUSINESS_TENANT_PATH`

    log.append("alice", "login", "u_1")

    ok, bad = log.verify_chain()
    assert ok is False

## 7. 风险与后续

| 项 | 风险 | 缓解 |
|---|---|---|
| 单进程 in-memory backend | 多 uvicorn worker 时状态不同步 | 已暴露 `IMDF_BUSINESS_*_PATH` 环境变量, 生产可切 JSONL 文件; 长期建议迁 Postgres |
| JSONL 无外部锁 | 多进程并发写可能丢行 | 单进程 OK, 多进程需 flock; 商用场景建议用 S3 / DB |
| 阶梯定价 hard-coded | 需后台配置界面 | `TieredPricing` 设计为可注入, 后续接管理端 |
| 审计链无 truncate 防护 | 文件无限增长 | JSONL 用 RotatingFileHandler 滚动可补, 当前 v1 不带 |
