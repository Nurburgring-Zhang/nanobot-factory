## 2. 步骤 1: 采集 Ingest (CSV)

**引擎**: `imdf.engines.ingestion_engine.IngestionEngine`

### 2.1 真实跑测

```python

# ❌ sqlite3.OperationalError: duplicate column name: id
```

### 2.2 原因

```python
# ingestion_engine.py:57
cursor.execute(f"CREATE TABLE IF NOT EXISTS [{table}] (id INTEGER PRIMARY KEY AUTOINCREMENT, _imported_at TEXT, {col_defs})")
#                                                    ↑ 已有 id                    ↑ 用户 CSV 又有 id
```

### 2.3 修复 (1 行)

```python
safe_cols = [(f"col_{c}" if c == "id" else c) for c in cols]
col_defs = ", ".join([f'"{c}" TEXT' for c in safe_cols])
```

## 7. 步骤 6: 分类 Classification (❌ BUG)

**引擎**: `imdf.engines.classification_engine.ClassificationEngine`

### 7.1 真实跑测

```python
engine = ClassificationEngine(db_path=":memory:")
engine._init_db()
rule = ClassificationRule(id="r1", name="is_animal", category="生物",
                          field="label", operator="contains", value="cat")
engine.rules[rule.id] = rule  # ← 只 set dict, 没写 DB

items = [{"label": "cat_white"}, {"label": "car_red"}, {"label": "cat_black"}]
out = engine.classify_batch(items)
# ❌ sqlite3.OperationalError: no such table: classification_rules
```

### 7.2 原因

- `db_path=":memory:"` 跨 connection 不共享
- `_init_db()` 在 conn_1 建表, `_load_rules()` 在 conn_2 查表 → 看不到
- 实际存 rule 走 `self.rules` dict, 没调 `add_rule()`

### 7.3 修复 (1 行)

```python
# 方案 A: shared cache
db_path = db_path or "file::memory:?cache=shared"

# 方案 B: 用单 conn
self._conn = sqlite3.connect(db_path, check_same_thread=False)
# 所有方法用 self._conn 而非 sqlite3.connect(...)
```
