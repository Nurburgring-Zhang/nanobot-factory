## 设计契约 (与 R2 7 验证模式对齐)

- **G 时间范围模式**: `dr: DateRangeParams = Depends()`,5 校验失败模式 (反序 / 超未来 / 跨度 > 365 / 起始 > 历史窗口 / preset=custom 缺 start 或 end) 全部走 HTTPException → 400。
- **F 粒度模式**: `Granularity = Query("day")`,Pydantic Literal 自动 422 拦截枚举外值。
- **D 维度模式**: `is_valid_dimension(dimension, scope="stats"|"monitor"|...)`,白名单外 → 400。
- **错误信息中文化**: 所有 detail 都含中文字段名,符合 G4。

## 已知限制

1. `pagination_compat.PaginationParams` 本轮未注入新端点 (R2-5 任务不主改分页端点)。
2. `ops_overview` 加的 `dimension` 参数当前未参与聚合,只做白名单校验;语义保留为后续可视化扩展。
3. canvas_web.py 顶层 `@app.exception_handler(Exception)` 仍会把未分类异常转为 500 (含 traceback)。本轮 W5 端点都不走那条路径 (都先 raise HTTPException 或 Pydantic 自动 422)。
