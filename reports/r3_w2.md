## 2. Findings (修复点)

### 2.1 模型权重 → 动态 (来自 /api/aesthetic/health)

`/api/aesthetic/health` 返回:

```json
{
  "success": true,
  "data": {
    "engine": "aesthetic_engine v3.1",
    "scoring_methods": { "heuristic": true, "llm_vision": false, "elo": true },
    "available_models": 0,
    "vision_models": 0,
    "elo": { "total_entries": 0, ... }
  }
}
```

3 个模型的 weight (45/30/25) 来自引擎 `MODEL_WEIGHTS` 常量,可用性由 `scoring_methods` 动态决定:
- Q-Align → `heuristic || llm_vision` (Pillow baseline 即视为可用)
- LAION / MUSIQ → `llm_vision` (需 vision-capable LLM)

每个模型条根据状态显示绿色 "就绪" 或橙色 "未启用" 标签。

### 2.2 6 维度评分 → 真实 (来自 /api/aesthetic/score)

`POST /api/aesthetic/score` 返回结构:

```json
{
  "success": true,
  "data": {
    "success": true,
    "overall_score": 8.5,
    "dimensions": {
      "composition": 8.5, "color": 7.0, "lighting": 6.5,
      "sharpness": 9.0, "content": 7.5, "creativity": 8.0
    },
    "models_used": ["q_align", "laion", "musiq"],
    "model_scores": { "q_align": 8.5, "laion": 7.5, "musiq": 6.0 },
    "confidence": "high"
  }
}
```

新版 `ac_renderDimensions(data)` 用 `AC_DIMS` 常量定义 6 维度 (`composition / color / lighting / sharpness / content / creativity`),**遍历响应实际数据**绘制百分比条,不再使用任何硬编码。

额外显示综合分、置信度 (`high / medium / low` → 绿/蓝/橙 tag) 和使用的模型列表。

### 2.3 Elo 排行榜 → 新增 (来自 /api/aesthetic/elo-ranking)

旧版 `ac_batchScore` 调用 `/api/aesthetic/elo-ranking` 但只 toast 数量,排行榜从未真正渲染。

新版 `ac_loadElo()` 在 `renderAestheticCenter` 启动时即拉取 Top 10,显示:
- 排名 (#1-#10)
- image_name (XSS 安全: `sanitizeHTML`)
- Elo rating
- W/L/D 战绩

空数据时显示空状态 ("评分后自动生成排行")。

## 8. 风险与限制

1. **prompt() 输入**:图片路径仍用 `prompt()` 而非 file picker。文件上传走 base64 需要后端新增 `image_base64` 字段(超出本任务范围)。
2. **未启用模型**: 当 `/health` 返回 `llm_vision: false` 时,LAION/MUSIQ 标记 "未启用",但仍展示 weight (源自引擎常量)。这是设计意图 — 让用户看到引擎理论支持的能力,而非实际可用性。
3. **批处理进度**: 批量评分无进度回调,如图片量大需后端先支持 SSE/WS。
