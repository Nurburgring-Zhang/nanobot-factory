## 二、metrics.py 设计要点

- **4 类指标**:HTTP (QPS / p50 / p95 / p99 / 错误率)、DB (查询总数 / 延迟 / 慢查询计数)、Cache (命中/未命中/写入/删除 + 延迟)、Process (RSS / uptime)
- **降级兼容**:`prometheus_client` 不可用时全部 stub 化,不抛异常
- **独立 CollectorRegistry**:不污染全局默认 registry
- **path normalization**:middleware 把 UUID / 长数字替换成 `{id}`,避免 cardinality 爆炸
- **公共 API**:`record_request` / `observe_db_query` / `cache_hit/miss/set/delete` / `render()`
- **兼容导出**:`engines.metrics` 通过 re-export 继续工作(不破坏 R1-R6 已有调用)

## 三、cache.py 设计要点

- **LRU 实现**:`OrderedDict` + `threading.Lock`,O(1) get/set,容量控制 + eviction 计数
- **Redis 可选**:环境变量 `IMDF_CACHE_REDIS_URL` 配置则走 Redis,否则本地 LRU
- **TTL 配置**:`IMDF_CACHE_LIST_TTL=300`(5min)/ `IMDF_CACHE_DETAIL_TTL=60`(1min)
- **失效钩子**:`invalidate_prefix()` 支持写后失效,`delete()` 显式删除
- **装饰器用法**:`@list_cache(ttl=300, key_prefix="projects:list")` 直接挂路由
- **统计**:`hits / misses / sets / deletes / evictions`,通过 `cache_hit()` 等函数上报 metrics

## 四、slow_query.py 设计要点

- **SQLAlchemy event**:`before_cursor_execute` / `after_cursor_execute` 拦截
- **阈值默认 200ms**:`install_slow_query_listener(engine, threshold_ms=200)`
- **三路输出**:Python logging (WARNING, `imdf.slow_query`)、JSONL 文件 (`logs/slow_queries.jsonl`)、Prometheus Counter (via metrics.observe_db_query)
- **参数脱敏**:不输出明文参数,避免敏感数据落盘
- **线程安全 + 单例**:同一 Engine 只装一次
