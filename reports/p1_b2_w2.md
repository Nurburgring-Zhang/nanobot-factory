## 四、关键技术决策

### 1. 旧 oss_upload() 用了 multipart, 后端实际是 JSON → 已修复
旧代码:
```js
fetch('/api/oss/upload?bucket=' + ..., { method: 'POST', body: fd })  // 422
```
新代码 (`oss_startUploads()`):
```js
const buf = await u.file.arrayBuffer();
const b64 = btoa(/* uint8 → binary string */);
await apiPost('/api/oss/upload', { key, content: b64, metadata: {...} });
```

### 2. 后端 query 返回的 objects 是 keys (string[]), 不是 file 对象
前端用 `_oss_synthFileFromKey()` 从 key 字符串合成完整 file 对象 (推断 bucket/ext/type/size/mtime), 用于表格渲染。

### 3. 单例状态用闭包常量 `_OSS_STATE`
切换页面后保留 filters/uploads/trash/customBuckets, 重新进入不用重传 API。pagination 不持久化 (每次重置到 page 1)。

### 4. 全局拖拽用防重入锁
```js
if (!window.__oss_dropzone_bound__) {
  window.__oss_dropzone_bound__ = true;
  oss_bindGlobalDropzone();
}
```

### 5. 所有 window.* 暴露 → 28 个函数
inline onclick 需要函数名在全局, 全部通过 `window.X = X` 显式挂载 (与 audio-tools.js / oss-storage.js 原版一致)。
