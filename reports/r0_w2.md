## 5. 关键设计决策

### 5.1 优先接入 AIRI 主类, fallback stub

新 router 优先调用 `backend/airi_digital_human.py` 已有的 `get_digital_human()` 单例:
- 若 AIRI 主类提供 `submit_generate_job` / `enqueue` 方法, 直接调用返回真实 `job_id`
- 否则 fallback 到与 `server_nanobot.py` 一致的 `dh_<md5[8]>_<uuid[6]>` stub job_id

这样 8900 ↔ 8898 行为对等, 但未来 AIRI 类补上 submit 方法时不需要再改 router。

### 5.2 路径解析 — sys.path 注入

`backend/imdf/engines/airi_digital_human.py` 需要 import 上一层 `backend/airi_digital_human.py`,
所以:
```python
_BACKEND_DIR = Path(__file__).resolve().parent.parent.parent  # backend/
if _BACKEND_DIR.is_dir():
    sys.path.insert(0, str(_BACKEND_DIR))
```

(初始版本误写成 `.parent.parent.parent / "backend"` → `backend/backend`, 已修正)

### 5.3 校验与错误响应

- `POST /digital-human/generate`: 未知 model → `HTTPException(400)` 返回结构化 body
- 所有端点 try/except 兜底返回 `{success:false, error:str(e)}`, 与 `server_nanobot.py` 一致
