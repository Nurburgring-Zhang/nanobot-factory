### 1.2 原代码三大问题 (审计前)

| 问题类型 | 表现 | 严重度 | 影响 |
|----------|------|--------|------|
| **类型契约缺失** | `const d = p.data as any;` (49 个文件) | HIGH | TS 类型系统失效,IDE 无法检查字段拼写错误 |

| **内嵌组件引用** | `from './ImageOpFrame'` / `from './useHasAutoOutput'` 等 7+ 个未实现模块 | CRITICAL | 同上,React 节点组件整体未编译 |

### 2.3 49 个 _node.tsx 文件批量改造

通过 Python 脚本 `refactor_nodes.py` 对所有 49 个节点统一做 3 件事:

1. **加 type 导入**:
   ```ts
   import type { NodeDataShape } from './types';
   import { mergeDefaultData } from './defaults';
   ```

2. **加 IO 契约 header**:
   ```ts
   /**
    * ─── R3-Worker-4 IO CONTRACT MARKER ────────────────────────────────
    * 类型 key : image
    * 组件     : ImageNode
    * Data in  : NodeDataShape<'image'>
    * Data out : imageUrl/imageUrls 写回 data
    * 默认值   : mergeDefaultData('image', p.data) → defaults.ts
    * IO 文档   : NODE_IO_CONTRACTS → 类型 'image'
    * onChange : useUpdateNodeData(id).update(patch)
    * ────────────────────────────────────────────────────────────────────
    */
   ```

3. **替换内嵌默认**:
   ```ts
   // 之前:
   const d = p.data as any;
   const width = d?.width || 1024;
   const fit = d?.fit || 'cover';
   
   // 之后:
   const d = mergeDefaultData('resize', (p.data as Partial<NodeDataShape<'resize'>>) || undefined);
   // width/fit 由 defaults.ts 统一提供
   ```

### 2.4 改造后的 onChange 数据流

每个节点通过 `useUpdateNodeData(id).update(patch)` 触发 onChange:

```ts
// 节点内部
const onField = useCallback(
  (key: keyof PortraitMeta, val: string) => {
    const next = { ...d, [key]: val };
    update({ ...next, prompt: buildPrompt(next) });  // ← onChange 入口
  },
  [update, d]
);
```

下游节点通过 xyflow 官方 `useNodesData(id)` 读取上游 data,实现真实数据流。

## 三、原则 (设计要点)

### 3.1 数据契约集中管理

所有节点的 data 形状集中在 `types.ts` 的 `NodeDataMap` 接口中,新增节点时只需:
1. 在 `NodeDataMap` 加 `<key, *Data>` 映射
2. 在 `defaults.ts` 加默认值
3. 在 `ALL_NODE_TYPES` 数组加类型 key
4. 在 `NODE_IO_CONTRACTS` 加文档元数据

### 3.2 默认值可配置

- 旧模式: `d?.width || 1024` (硬编码在 JSX 中,需扫文件)
- 新模式: `mergeDefaultData('resize', p.data)` (集中 `defaults.ts`)

### 3.3 onChange 真实数据流

- 旧: 直接 `useState` 维护本地副本,与 xyflow store 不同步
- 新: `useUpdateNodeData(id).update(patch)` 浅合并到 xyflow store,触发所有监听节点重渲染

### 3.4 渐进式类型注解

保留 `Partial<NodeDataShape<T>>` 允许部分配置,避免 strict 模式过度严格导致 refactor 工作量爆炸。
