## 设计决策

1. **保留 R3 函数式架构, 不转换为 Vue 3 SFC**: 任务描述用了 Vue 3 SFC 语法, 但 `index.html` 通过
   `<script src>` 加载 36 个页面, `app.js` 通过 `window.renderXxx` 全局调用, 切到 SFC 需要
   改打包链/路由/全部依赖, 超出本任务范围。新代码用 `let AL/TC = { ... }` 集中状态 + 一组顶层
   `function al_xxx / tc_xxx`, 风格与 `datasets.js` (424 行) / `template-pipeline.js` (424 行) 一致。

2. **transfer 适配 share-link 而非 file-transfer**: 任务描述的 "传输任务 (upload/download/copy)"
   后端不存在。已按实际 API 重命名: "资源路径 / 创建者 / 过期时间 / 状态 / 下载次数" 替代
   "源/目标/进度/大小"。"批量操作" 改为 批量撤销 + 批量永久删除。

3. **客户端二次过滤 level/status/type**: 后端 `audit_routes.py` 不支持 level/status/type 参数,
   客户端按 status_code 与 is_active/expires_at 推算。这样过滤完全无服务往返, 切换即时。

4. **自动刷新 + 倒计时可见**: 30s 自动刷新 + 1s 倒计时指示 (`🔄 28s 后刷新`) 让用户清楚状态,
   标签页切回可见时立刻重置倒计时。`beforeunload` 清理防止内存泄漏。

5. **批量操作进度反馈**: 循环 N 次 DELETE 时, 用 `showGlobalLoading(\`撤销中 (${i+1}/${N})\`)`
   实时反馈, 而非无指示阻塞。失败计数分别返回。
