六、需要修复的严重问题
=============================================================================
🔴 P0 - 崩溃性缺陷:
  1. /api/aesthetic/* 全模块 (8端点) - 返回500内部错误而非正常响应
     → POST score/score-batch, GET elo-ranking/elo-stats等全部崩溃
     → 可能缺少依赖模块(模型文件/数据库表)
  2. /api/aesthetic/elo-entry/{image_id} - bad_params触发连接中断(status=0)
  3. /api/drama/episode/{episode_id} - bad_params触发连接中断(status=0)
  4. /canvas/element/{element_id} - bad_params触发连接中断(status=0)

🟡 P1 - 参数验证缺失:
  272个端点(55.9%)在收到无效参数时返回200而非400/422
  → 缺乏统一的参数验证中间件
  → 建议添加全局输入验证层(FastAPI/Pydantic)

🟡 P1 - 3个端点路径参数存在注入风险:
  /api/aesthetic/elo-entry/{image_id}, /api/drama/episode/{episode_id},
  /canvas/element/{element_id} - SQL注入类参数导致连接中断

🟢 P2 - 频率限制:
  /auth/login 存在429限流(测试中多次调用触发)

🟢 P2 - 认证一致性:
  部分端点(6个)在no_auth场景返回200但openapi标注需要认证
